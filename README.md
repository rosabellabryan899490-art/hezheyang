# 打卡计划（PWA）

把一份"按天拆解"的学习计划表变成每天的打卡任务。**没完成的任务会自动累计到后面，补完为止。**

- 手机网页应用（PWA）：浏览器打开 → 添加到主屏幕 → 图标和普通 App 一样，离线可用
- 数据只存在本机浏览器（localStorage），无需注册、无需联网
- 模板驱动：导入模板 JSON → 选开始日期 → 自动铺到每一天；模板可导出、微信发给研友
- 内置示例模板：2027 湖科大学科物理考研计划（152 天 / 488 个任务，2026-07-20 ~ 12-18，考试 12-19）

## 文件说明

| 文件 | 作用 |
|---|---|
| `index.html` | 应用页面与全部样式（今日 / 日历 / 统计 / 模板 四页） |
| `app.js` | 应用逻辑（打卡、积压累计、统计、模板导入导出、备份） |
| `template-kaoyan.js` | 内置示例模板生成器 |
| `manifest.webmanifest` / `sw.js` / `icon.svg` | PWA 配置（主屏幕图标、离线缓存） |
| `server.js` | 零依赖本地静态服务器（预览用） |
| `test-app.js` | 冒烟测试（`node test-app.js`） |

## 本地预览（电脑）

```bash
node server.js
# 打开 http://localhost:8080
# 手机与电脑连同一 WiFi 时，也可用控制台打印的局域网地址访问
```

## 部署到 GitHub Pages（手机随时可用 + 可分享链接）

1. 打开 <https://github.com> 注册/登录。
2. 右上角 `+` → **New repository**：名字填 `daka-plan`，选 **Public**，其余默认，点 Create。
3. 仓库页 → **Add file → Upload files**，把本项目文件拖进去上传
   （`index.html`、`app.js`、`template-kaoyan.js`、`manifest.webmanifest`、`sw.js`、`icon.svg`、`README.md`；
   **不要上传 `api.txt`**——那是私人密钥）。拉到底部点 **Commit changes**。
4. 仓库 **Settings → Pages** → Source 选 **main** 分支、**/(root)** → Save。
5. 等 1~2 分钟，访问 `https://<你的用户名>.github.io/daka-plan/` 即上线。

## 手机使用

- **安卓 Chrome**：打开网址 → 右上角菜单 → **添加到主屏幕**。
- **iPhone Safari**：打开网址 → 分享按钮 → **添加到主屏幕**。
- 首次打开后：选择内置模板 → 开始日期 `2026-07-20` → 考试日期 `2026-12-19` → 创建计划，开始打卡。
- 打卡数据只存在当前手机浏览器里，**清除浏览器数据会丢失**；每月在「模板 → 数据备份」导出一次备份文件更稳妥。

## 制作自己的模板（发给研友）

模板是一个 JSON 文件，**只存"第 N 天"，不存具体日期**——对方导入后选自己的开始日期即可铺开。

1. 用 Excel 列四列：`第几天 | 任务内容 | 科目 | 预计分钟`；
2. 让 AI 转成下面的格式并保存为 `.json` 文件：

```json
{
  "name": "模板名",
  "author": "你的名字",
  "days": [
    { "day": 1, "tasks": [
      { "title": "背单词40分钟", "subject": "英语二", "minutes": 40 }
    ] }
  ]
}
```

3. 微信把 `.json` 发给对方 → 她在 App「模板 → 导入模板文件」选择即可。
4. 内置模板也可在「模板」页导出，作为参考样板。

## 规则说明

- 逐条勾选完成；点「跳过」可将某条任务移出统计（可恢复）。
- 昨天及更早没完成的任务，自动汇总到「今日」页的**积压补做**区，按原日期分组，补完为止。
- 未来的任务不可提前打卡；连续打卡天数按"当天至少完成 1 项"计。
